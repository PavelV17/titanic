## Что было сделано в проекте

### 1. Анализ данных

Был проведён исследовательский анализ данных Titanic:

- проверены размеры train и test;
- изучены типы данных и пропуски;
- проверен баланс целевого признака `Survived`;
- изучена выживаемость в зависимости от пола, класса, возраста, стоимости билета и порта посадки;
- построены графики и корреляционная матрица.

Основные наблюдения:

- женщины выживали значительно чаще мужчин;
- пассажиры первого класса выживали чаще пассажиров второго и третьего классов;
- стоимость билета и класс пассажира связаны с вероятностью выживания;
- семейные связи и обращение в имени содержат дополнительную информацию.

---

### 2. Feature Engineering

Были созданы новые признаки:

- `FamilySize` — общее количество членов семьи на борту;
- `IsAlone` — путешествовал ли пассажир один;
- `CabinKnown` — известен ли номер каюты;
- `TitleGrouped` — обращение пассажира, извлечённое из имени: Mr, Mrs, Miss, Master или Rare.

После добавления новых признаков accuracy Logistic Regression выросла примерно с:

- `0.7969` без новых признаков;
- до `0.8283` с новыми признаками.

Это показывает, что feature engineering улучшил качество модели.

---

### 3. Предобработка данных

Для числовых признаков использовались:

- заполнение пропусков медианой;
- стандартизация через `StandardScaler`.

Для категориальных признаков использовались:

- заполнение пропусков наиболее частым значением;
- преобразование категорий через `OneHotEncoder`.

Предобработка и модель были объединены в единый `Pipeline`.

---

### 4. Классические модели

Были проверены следующие модели:

#### Logistic Regression

Линейная модель бинарной классификации. Рассчитывает вероятность выживания пассажира на основе взвешенной суммы признаков.

Результат после feature engineering:

- mean accuracy: `0.8283`;
- std: `0.0101`.

Модель показала хороший и стабильный результат.

#### K-Nearest Neighbors

Класс пассажира определяется по классам ближайших похожих пассажиров.

Результат:

- mean accuracy: `0.8204`;
- std: `0.0371`.

Модель оказалась менее стабильной, поскольку KNN чувствителен к расположению точек и выбранному количеству соседей.

#### Decision Tree

Модель последовательно разделяет данные с помощью правил вида:

- `Sex == female`;
- `Pclass <= 2`;
- `Age <= значение`.

Результат:

- mean accuracy: `0.8137`;
- std: `0.0222`.

Одиночное дерево показало сравнительно слабый результат и склонность к переобучению.

#### Random Forest

Ансамбль большого количества деревьев решений. Каждое дерево обучается на своей выборке и части признаков, после чего результаты деревьев объединяются голосованием.

Результат:

- mean accuracy: `0.8328`;
- std: `0.0099`.

Random Forest оказался лучше и стабильнее одиночного Decision Tree.

---

### 5. Градиентный бустинг

Были проверены три бустинговые модели:

- CatBoost;
- LightGBM;
- XGBoost.

В градиентном бустинге деревья строятся последовательно. Каждое следующее дерево пытается исправить ошибки предыдущих деревьев.

#### CatBoost

CatBoost умеет работать с категориальными признаками напрямую, без обязательного One-Hot Encoding.

Для CatBoost использовались:

- `Pclass`;
- `Sex`;
- `Embarked`;
- `TitleGrouped`

как категориальные признаки.

#### LightGBM

LightGBM — быстрый алгоритм градиентного бустинга по деревьям. Перед моделью использовался общий preprocessing pipeline.

#### XGBoost

XGBoost также строит последовательность деревьев и использует регуляризацию для контроля переобучения.

Все бустинговые модели были проверены через стратифицированную кросс-валидацию.

---

### 6. Deep Neural Network

Нейронные сети были реализованы на PyTorch.

#### DNN-1

Простая сеть:

`вход → 64 нейрона → ReLU → выход`

Validation accuracy:

`0.8101`

#### Более глубокие сети

Далее проверялись архитектуры:

- `128 → 64`;
- `64 → 32`;
- `256 → 128`.

Увеличение сети до `256 → 128` не улучшило результат. На небольшом датасете Titanic слишком большая сеть быстрее переобучается.

#### Batch Normalization и Dropout

В сеть были добавлены:

- `BatchNorm1d`;
- Dropout со значениями `0.1`, `0.3` и `0.5`.

BatchNorm стабилизирует значения между слоями.

Dropout во время обучения случайно отключает часть нейронов и помогает снижать переобучение.

#### Функции активации

Были проверены:

- ReLU;
- LeakyReLU;
- SiLU.

Лучший результат среди проверенных активаций показала SiLU.

#### Оптимизаторы

Были проверены:

- Adam;
- SGD с momentum.

Adam показал более высокий результат.

#### Scheduler

Использовался:

`CosineAnnealingLR`

Он постепенно изменяет learning rate по косинусному расписанию во время обучения.

#### Параметры обучения

Проверялись разные:

- learning rate;
- batch size;
- количество эпох;
- размеры скрытых слоёв;
- значения Dropout.

Лучшей стала модель DNN-11:

- архитектура: `128 → 64`;
- активация: SiLU;
- BatchNorm: да;
- Dropout: `0.3`;
- optimizer: Adam;
- scheduler: CosineAnnealingLR;
- learning rate: `0.001`.

Результат пятифолдовой проверки:

- mean accuracy: `0.8361`;
- std: `0.0112`.

Embedding для категориальных признаков не выполнялся, поскольку это было дополнительное задание со звёздочкой.

---

### 7. Ансамбли

Были реализованы три типа ансамблей:

- Averaging;
- Soft Voting;
- Stacking.

#### Averaging

Для каждой модели рассчитывалась вероятность выживания:

- Logistic Regression;
- Random Forest;
- LightGBM;
- XGBoost.

Затем вероятности усреднялись:

`средняя вероятность = сумма вероятностей моделей / количество моделей`

Если средняя вероятность была не меньше `0.5`, прогнозировался класс `1`.

Averaging был реализован вручную.

#### Soft Voting

Soft Voting делает почти то же самое, что Averaging, но через готовый `VotingClassifier`.

Каждая модель выдаёт вероятность класса `1`, после чего вероятности усредняются.

В ансамбль входили:

- Logistic Regression;
- Random Forest;
- LightGBM;
- XGBoost.

Результат:

- mean accuracy: `0.8440`;
- std: `0.0187`.

Soft Voting показал лучший средний результат среди всех проверенных моделей.

#### Stacking

В Stacking базовые модели сначала делают прогнозы:

- Logistic Regression;
- Random Forest;
- LightGBM;
- XGBoost.

Эти прогнозы становятся новыми признаками для финальной метамодели.

Были проверены две метамодели:

- Logistic Regression;
- RidgeClassifier.

Результаты:

- Stacking Logistic: `0.8417 ± 0.0121`;
- Stacking Ridge: `0.8417 ± 0.0149`.

Обе модели показали одинаковую среднюю accuracy, но Stacking Logistic оказался стабильнее.
